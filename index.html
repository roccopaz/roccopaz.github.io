AI Content Clipping Pipeline

Google APIs · AI Agents · Automation

Built a fully automated video content pipeline that takes raw footage from Google Drive and produces ready-to-publish YouTube Shorts and TikTok clips — with AI handling the analysis, selection, and packaging.

🔄 How the Pipeline Works
Step 1
Upload — Raw source videos dropped into Google Drive Incoming Source Clips folder via Drive API
Step 2
AI Analysis — Agent uses FFmpeg + Whisper to extract frames, detect silence, transcribe audio, and generate contact sheets
Step 3
Clip Selection — AI recommends best timestamps for hooks, highlights, and viral moments
Step 4
Export — Draft clips rendered with FFmpeg, auto-uploaded to Draft Exports folder in Drive
Step 5
Tracking — Google Sheets logs every clip (status, timestamps, notes) via Sheets API

Secure AI Service Architecture

Security · OAuth · DevSecOps

When deploying an always-on AI system that interacts with real accounts and APIs, security cannot be an afterthought. I designed a layered security architecture using the same principles found in enterprise DevSecOps environments.

🛡️ Security Layers
LAYER 1
Tailscale VPN (Zero Trust Networking)
The VPS runs Tailscale — a WireGuard-based mesh VPN. All admin access tunnels through Tailscale, meaning no SSH port is exposed to the public internet. The server is invisible to external scanners. Eliminates brute-force attacks at the network level.
LAYER 2
Service Identity Isolation
Dedicated Google account created exclusively for the AI system. Personal email and data are never exposed to the AI runtime — even if service credentials were compromised, the blast radius is fully contained.
LAYER 3
Scoped OAuth Permissions (Least Privilege)
Drive, Docs, and Sheets APIs each authorized with minimum required scopes. Gmail intentionally excluded — the AI cannot read or send personal email by design.
LAYER 4
Encrypted Credential Storage
OAuth tokens stored via encrypted keyring backend on the VPS — password-protected, never plaintext, never committed to version control. API keys (Anthropic, OpenAI) stored as environment variables, isolated from codebase.
📚 Concepts Demonstrated
Zero Trust Networking Tailscale VPN WireGuard Least Privilege Service Identity Isolation OAuth 2.0 Blast Radius Containment Secrets Hygiene Defense in Depth DevSecOps

OpenClaw VPS AI System

AI · DevOps · Automation

Deployed and operate a production multi-agent AI system on a Linux VPS — an always-on infrastructure where specialized AI agents collaborate autonomously to handle research, content creation, coding, and orchestration tasks.

🏗️ Architecture
  • OpenClaw Gateway — Node.js-based orchestration runtime managing agent lifecycles, routing, and memory
  • 4 Specialized AI Agents — Each with distinct roles, models, memory, and channel bindings
  • Discord as Command Interface — Agents receive and respond to tasks through dedicated Discord channels
  • Persistent Memory System — Agents retain context across sessions via structured markdown memory files
  • Cron Scheduler — Automated recurring tasks (trend reports, content packs, reminders) on precise schedules
